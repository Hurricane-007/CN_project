HTTP Range Requests

Video streaming uses Range: bytes=start-end headers. The server responds with 206 Partial Content, delivering only the requested chunk — enabling seeking, buffering, and adaptive delivery.

GET /stream/abc123 HTTP/1.1
Range: bytes=0-1048575

HTTP/1.1 206 Partial Content
Content-Range: bytes 0-1048575/15728640
Content-Length: 1048576
🗄️

Cache-Control Headers

Static video files are served with Cache-Control: public, max-age=3600. Dynamic data (user cache list) uses no-store to prevent stale data.

Video stream:
Cache-Control: public, max-age=3600

User cache API:
Cache-Control: no-store
🔐

Stateless Auth (JWT)

Authentication uses JSON Web Tokens stored in HTTP cookies. The server validates tokens on each request without maintaining server-side sessions — demonstrating stateless protocol design.

Cookie: token=eyJhbGci...
Authorization: Bearer <jwt>

Payload: { username, exp }
Signed with HMAC-SHA256
📶

Client–Server Architecture

Server binds to 0.0.0.0:3000, making it accessible from all network interfaces. Multiple clients on the same LAN/hotspot can simultaneously stream from the same server.

Server: 0.0.0.0:3000
Clients: any device on LAN

Access: http://<server-IP>:3000
Protocol: TCP/IP + HTTP/1.1
